Generate a **cryptographically verifiable publish receipt** for every CI/CD release.

Each pipeline run produces a **signed proof capsule anchored in the GetIntegrity tamper-evident ledger**, enabling independent verification of software releases, artifacts, and deployment events.

This allows teams to **prove when and how a release occurred** without operating validator infrastructure or additional security tooling.

Designed for **DevOps, DevSecOps, and security teams** that require verifiable integrity across build pipelines.

---

# Why This Matters

Modern software supply chains rely on CI/CD pipelines — yet most builds leave **no independently verifiable record** of what was released and when.

This Action solves that gap by generating a **publicly verifiable proof receipt** for every release event.

Use cases include:

- **Release integrity verification**
- **Supply-chain transparency**
- **Audit-ready deployment evidence**
- **Cryptographic event lineage tracking**
- **Independent release verification**

# What This Action Generates

Each workflow run produces **verifiable integrity artifacts**:


receipt.json
receipt.sha256
receipt.pdf


These artifacts provide:

| Artifact | Purpose |
|--------|--------|
| `receipt.json` | Canonical machine-readable integrity record |
| `receipt.sha256` | Cryptographic hash for **offline verification** |
| `receipt.pdf` | Human-readable receipt for audits and compliance |

Artifacts can be stored with:

- release artifacts
- deployment records
- compliance documentation
- audit evidence packages
- supply-chain security archives

## Usage

Add the Action to your workflow.

```yaml
name: Publish Receipt

on:
  push:
    branches: [ main ]

jobs:
  publish-proof:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Generate Publish Receipt
        id: publish
        uses: GetIntegrityAPI/proof-of-publish@v1
        with:
          api_key: ${{ secrets.GI_API_KEY }}

      - name: Show Verification Receipt
        run: |
          echo "Proof ID: ${{ steps.publish.outputs.proof_id }}"
          echo "Verification URL: ${{ steps.publish.outputs.receipt_url }}"

      - name: Publish Verification Summary
        run: |
          echo "## GetIntegrity Publish Receipt" >> $GITHUB_STEP_SUMMARY
          echo "" >> $GITHUB_STEP_SUMMARY
          echo "**Proof ID:** ${{ steps.publish.outputs.proof_id }}" >> $GITHUB_STEP_SUMMARY
          echo "" >> $GITHUB_STEP_SUMMARY
          echo "**Verification URL:**" >> $GITHUB_STEP_SUMMARY
          echo "${{ steps.publish.outputs.receipt_url }}" >> $GITHUB_STEP_SUMMARY
```

This automatically attaches a **verifiable integrity receipt** to each CI/CD pipeline execution.

## Verifying a Proof

Proofs generated by this Action can be verified in two ways.

### Direct Proof Verification


https://getintegrityapi.com/verify/
<proof_id>


Example:


https://getintegrityapi.com/verify/p_7c31b9
...


### Online Verification Tool

Developers can also verify proofs using the public verification interface:


https://getintegrityapi.com/#verify

## Adding Your API Key

Store your API key as a GitHub Secret.

1. Open **Repository Settings**
2. Navigate to **Secrets → Actions**
3. Add a new secret:


GI_API_KEY


Then reference it in your workflow:

```yaml
api_key: ${{ secrets.GI_API_KEY }}

```
